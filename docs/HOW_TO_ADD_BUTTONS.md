# 如何在页面中添加按钮配置

## 🎯 快速开始

### 第一步：打开页面设计器

1. 访问页面管理页面
2. 找到你想要添加按钮的页面
3. 点击"设计"或"编辑"按钮，进入页面设计器

### 第二步：选择工具栏区域

1. 在中间的设计画板中，你会看到多个区域：
   - 📝 查询区
   - 🔧 **工具栏** ← 点击这个区域
   - 📊 内容区
   - 📋 树形区

2. 点击"工具栏"区域，它会被高亮显示

### 第三步：从组件库选择按钮

左侧会显示**组件库面板**，包含：

**通用组件库**（系统预置的按钮）：
- ➕ 新增（btn_add）
- ✏️ 编辑（btn_edit）
- 🗑️ 删除（btn_delete）
- 👁️ 查看（btn_view）
- 📥 导出（btn_export）
- 📤 导入（btn_import）
- 🔍 查询（btn_search）
- 🔄 重置（btn_reset）
- 🔄 刷新（btn_refresh）
- ✔️ 提交（btn_submit）
- ❌ 取消（btn_cancel）
- 💾 保存（btn_save）

**添加按钮的两种方式：**

1. **点击添加**：
   - 在按钮列表中点击想要的按钮
   - 按钮会自动添加到工具栏

2. **拖拽添加**：
   - 按住按钮，拖拽到右侧的预览区域

### 第四步：保存页面

1. 点击顶部"保存"按钮
2. 系统会保存按钮配置

### 第五步：查看效果

1. 访问页面路由
2. 工具栏会显示所有添加的按钮
3. 点击按钮测试功能

## 🛠️ 按钮管理

### 创建自定义按钮

1. 访问**按钮管理**页面
2. 点击"新增按钮"
3. 配置按钮属性：
   - 按钮名称
   - 按钮编码
   - 组件分类（通用/业务）
   - 组件标签
   - 按钮类型
   - 图标
   - 动作类型
   - 权限标识

4. 保存后，按钮会出现在组件库中

### 通用按钮 vs 业务按钮

**通用按钮库**：
- 系统预置，可在所有页面复用
- 适用于常见操作（增删改查、导入导出等）
- 修改后全局生效

**业务按钮库**：
- 用户自定义，用于特定业务场景
- 按业务领域分类
- 灵活性更高

## 🔍 故障排查

### 问题：页面设计器中没有看到组件库面板

**解决方案：**
1. 确保选择了"工具栏"区域
2. 左侧会自动显示组件库面板
3. 如果没有显示，刷新页面

### 问题：按钮添加后页面不显示

**解决方案：**
1. 检查是否保存了页面
2. 刷新页面
3. 打开浏览器控制台查看错误信息

### 问题：按钮点击没有反应

**解决方案：**
1. 检查按钮的"动作类型"配置
2. 配置API接口地址
3. 检查权限配置

## 📝 示例：创建用户管理页面

### 1. 创建页面

- 页面名称：用户管理
- 页面编码：user_manage
- 页面类型：列表页

### 2. 配置内容区

- 选择"内容区"
- 从组件库选择"通用数据表格"

### 3. 配置工具栏按钮

- 选择"工具栏"区域
- 添加按钮：新增、编辑、删除、导出

### 4. 保存并发布

- 点击"保存"
- 点击"发布"
- 访问页面路由查看效果

---

## 🎓 最佳实践

1. **使用通用按钮**：优先使用通用组件库中的按钮
2. **合理命名**：按钮编码要有意义，如 `btn_add_user`
3. **权限控制**：为按钮设置权限标识
4. **标签分类**：使用标签便于管理和查找
5. **测试验证**：保存后立即测试按钮功能
